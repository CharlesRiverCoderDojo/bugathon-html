So, what's debugging, and why is it important?

Debugging is the process of looking through code very carefully to find where the code isn't written correctly. With HTML code, an example of an error that you may look for is that all tags that need to be closed are actually closed. More of types of errors later...

With programming, this skill comes in handy on a regular basis. As you understand the errors in your code more, you will improve on your programming skills, have a deep understanding of how to properly write code, and be able to write code faster.

For this exercise, follow your instinct, and if something just "feels" wrong in the code, look into it a bit deeper. Understand why the code is written in a certain way and determine if there is a problem with it. Importantly, you should try to understand WHY there could be a problem with the code. Use the internet to find answers to the questions that will come up during this exercise. One helpful site is W3Schools. You can also check out StackOverflow as well.

Here are some tips on looking for bugs:

  •     Pay close attention to HTML tags that must be closed
  •     For self-closing HTML tags, make sure the syntax is correct
  •     Ensure that an HTML element is being implemented correctly (hint, hint the list item tag)
  •     With styling, make sure that an element is being selected as a class or an id
  •     Check that styling of an element isn't getting overwritten by an element lower in the cascade
  •     Open up your web inspector (option + command + i on a Mac, and control + shift + i on a PC) to directly update CSS while in the browser
  •     If you're interested in learning about Font Awesome, check out this link here: Font Awesome Examples

Hello, ninja. For your extra challenge, you will be learning a bit about algorithms.

So, what exactly is an alogrithm?

To say exactly what Wikipedia says- it's a self-contained sequence of actions to be performed. Huh?? Basically, it's a series of steps that you take to reach a very specific goal. So, don't worry about content or styling in this extra challenge- think about the way your code functions when a user interacts with it.


Think about it- your life is a series of algorithms.


For example, you have an algorithm you follow every time you brush your teeth. Go into the bathroom, grab your toothbrush, turn on the water, wet your toothbrush, turn off the water, put toothpaste on your toothbrush, brush those beautiful teeth, etc. It's as easy as that! Well, easy may sometimes be an overstatement... Oy...


Algorithms can get pretty complicated, but they're not always that complicated. Take a few minutes to think about some of the algorithms that dominate your life. What is your algorithm for having dinner? What is the algorithm you follow just to turn on your computer?


Algorithms are very helpful in programming. Like, do you see the ‹em› tag and underline class that I'm using to emphasize how helpful they are?? They are the recipes that you can use to create your fantastic programs. They will help you build the foundation of code that you need for your site and make it applicable to many different situations. For example, if Laura and Kate have the same tooth-brushing routine, you can use that one tooth-brushing algorithm to represent that process for both people. A more generic example is that you can use an algorithm to calculate the number of days a person has been alive since their date of birth. So, you would take any one person's date of birth and figure out how many days that person has existed since today's date. Or you can sort a random series of numbers from the smallest to the largest. What would be an interesting algorithm that you could create based on your life?


By creating an algorithm, you will save time programming, and your code will be D.R.Y.- Don't Repeat Yourself. If you're interested, look up DRY programming.
