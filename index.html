This isn't meant to be easy, but it will help you build your debugging skills. So, what's debugging, and why is it important? Well, it's very rare that all of the code you create will be free of errors, or that code that you work with as a developer will be free of errors. That's why it is important to train yourself to recognize errors in code and understand why that block of code needs to be changed. That's the essence of debugging- find an error in the code that you're working with and fix it.


For this exercise, follow your instinct, and if something just "feels" wrong in the code, figure out why. Understand why the code is written in a certain way and fix it if you find a problem with it. Try turning off or disabling any linter packages that may be enabled in your text editor of choice. If you run into questions or are having trouble, you should search the internet for answers to your questions. One helpful site is W3Schools. You can also check out StackOverflow as well.

Here are some tips on looking for bugs:

  •     Pay close attention to HTML tags that must be closed
  •     For self-closing HTML tags, make sure the syntax is correct
  •     Ensure that an HTML element is being implemented correctly (hint, hint the list item tag)
  •     With styling, make sure that an element is being selected as a class or an id
  •     Check that styling of an element isn't getting overwritten by an element lower in the cascade
  •     Open up your web inspector (option + command + i on a Mac, and control + shift + i on a PC) to directly update CSS while in the browser
  •     If you're interested in learning about Font Awesome, check out this link here: Font Awesome Examples
