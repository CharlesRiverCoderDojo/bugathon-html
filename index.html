So, what's debugging, and why is it important?

Debugging is the process of looking through code very carefully to find where the code isn't written correctly. With HTML, an example of an error that you may look for is that all tags needing to be closed are actually closed. More of types of errors later...

With programming, this skill comes in handy on a regular basis. As you understand the errors in your code more, you will improve on your programming skills, have a deep understanding of how to properly write code, and be able to write code faster.

For this exercise, follow your instinct. If something just "feels" wrong in the code, look into it a bit deeper. Understand why the code is written in a certain way and determine if there is a problem with it. Most importantly, you should try to understand WHY there could be a problem with the code. Use the internet to find answers to the questions that will come up during this exercise. One helpful site is W3Schools. You can also check out StackOverflow as well.

Here are some tips on looking for bugs:

  • Pay close attention to HTML tags that must be closed
  • For self-closing HTML tags, make sure the syntax is correct
  • Ensure that an HTML element is used in the correct order (like with <td> and <tr> tags)
  • Make sure that image paths are correct or they will not work (look up relative and absolute paths)
